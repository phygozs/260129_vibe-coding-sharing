## 🎯 项目介绍

这是一个基于 Streamlit 的互动演示应用，展示了 AI-Coding 的核心理念、方法选择和实践步骤，包括：

- 🎯 **核心理念**：AI-Coding 的甜蜜区与三条警示线
- 📊 **AI-Coding 的方式**：三种方式的核心差异与选择标准
- 💬 **Prompt-Coding**：一句话让 AI 自己跑（图书馆预约展示页面案例）
- 🔧 **Vibe-Coding 1.0**：按你定的规则跑（锁目标+拆小步）
- 👥 **分组研讨**：实战练习环节
- 🚀 **Vibe-Coding 2.0**：规则从 prompt 中拆出，形成可复用判断
- ✅ **核心金句**：三种方法的总结与行动建议

## 📦 快速启动

### 方式一：一键启动（推荐）

```bash
chmod +x start.sh
./start.sh
```

### 方式二：手动启动

1. 安装依赖：
```bash
pip3 install -r requirements.txt
```

2. 启动应用：
```bash
streamlit run ai_coding_demo.py
```

3. 访问应用：
打开浏览器访问 `http://localhost:8501`

## 🔧 端口配置

默认端口为 **8501**（适配云平台健康检查）。

**当前运行状态**: 
- ✅ 已在8501端口运行
- 🔄 替代了旧版应用（260114-vibe_coding分享）

如需修改端口，可以：
1. 编辑 `.streamlit/config.toml`：
```toml
[server]
port = 你的端口号
```

2. 或启动时指定端口：
```bash
python3 -m streamlit run ai_coding_demo.py --server.port 你的端口号
```

## ❓ FAQ

### Q: 为什么用 8501 端口？
A: 云平台健康检查默认访问 8501，使用该端口便于通过部署校验。

### Q: 如何修改内容？
A: 直接编辑 `ai_coding_demo.py`，所有内容都在对应的章节 if/elif 块中。

### Q: 可以导出为静态网页吗？
A: Streamlit 应用需要运行时环境，建议部署到 Streamlit Cloud 或服务器。

### Q: 依赖安装失败怎么办？
A: 尝试使用国内镜像源：
```bash
pip3 install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```
